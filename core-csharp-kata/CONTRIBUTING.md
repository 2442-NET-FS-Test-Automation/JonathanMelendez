# Contributing Guidelines

Thank you for contributing to the Store Kata project.

## Team

This project is maintained by a two-person team.
- qJona65p
- FraJimVit

## Workflow

- Commit changes with clear and concise messages.
- Review code changes with at least one team member when possible.

## Coding Standards

- Follow standard C# naming conventions.
- Keep business logic separated from UI code.
- Prefer small, reusable methods over duplicated code.
- Add comments only when the intent of the code is not immediately clear.

## Testing

- Verify that the application builds successfully before pushing changes.
- Test affected features manually through the console application.

## Questions

If you have questions about the project structure or implementation details, contact the team.
